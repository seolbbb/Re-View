
# 핵심 아이디어
"마우스가 움직이는 구간을 우선적으로 수집하면, 
각 픽셀 위치에서 마우스가 없는 프레임이 과반수가 됨"

## Option 1: 움직임 감지 기반 수집 (추천 ⭐)

1. [t-2초 ~ t+4초] 구간에서 프레임 간 차이(Diff) 계산
2. Diff가 큰 프레임 = 마우스가 움직이는 순간
3. 움직임이 큰 프레임을 우선적으로 수집 (예: 상위 60%)
4. Temporal Median 계산

장점: 마우스가 움직이는 순간 = 각 픽셀에서 마우스가 없는 확률 증가 정적 마우스 문제 해결
단점: 계산량 약간 증가

## Option 2: 장기 구간 수집 (단순하지만 효과적)
현재: [t-2초 ~ t+4초] = 6초
변경: [t-5초 ~ t+10초] = 15초

장점:구현 단순 (파라미터만 변경)
긴 시간 동안 마우스가 한 곳에만 있을 확률 낮음
단점: 슬라이드 혼합 위험 증가, 처리 시간 증가

## Option 3: 다중 시점 샘플링 (고급)
1. 현재 슬라이드 전체 구간에서 무작위로 프레임 수집
   예: 슬라이드 1이 [0~300초]라면, 이 구간에서 랜덤 50프레임
2. Temporal Median 계산
장점: 슬라이드 전체에서 마우스 위치가 다양함 → 제거 확률 극대화, 슬라이드 순수성 100% 보장
단점: 슬라이드 경계 감지 필요 (복잡)


## Option 4: 에지 기반 마스킹 (컴퓨터 비전 기법)
1. 프레임에서 에지(Edge) 검출
2. 에지가 강한 영역 = 슬라이드 내용 (텍스트, 수식)
3. 에지가 약한 영역 = 마우스 포인터 (단색)
4. 에지 강도로 가중치를 준 Weighted Median
장점: 이론적으로 가장 정확, 슬라이드 내용 보존 + 마우스 제거
단점: 구현 복잡도 높음
